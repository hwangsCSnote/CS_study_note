# NW TCP 4계층 / PDU

> **IP**
> 

순서에 구애받지 않고 되는데로 패킷을 전달

때문에 손실이 나도 채워지지 않고 빠르게 전달하기만 한다

> **TCP**
> 

순서대로 패킷을 전달한다

손실이 나지않는 대신에 속도가 IP에 비해서 느리다

> **UDP**
> 

IP와 비슷하다 순서에 구애받지 않고 빠르게 전달하는 것이 목표이다

다만 체크섬 방식으로 좀 더 손실에 대비한다

TCP보다 빠르다.

그렇기에 실시간 동영상에서 사용한다

> **TCP/IP 4계층**
> 

실제 인터넷 구조 모델이 아니라 실전에서 사용중이다

OSI 7계층은 권고문, 이론같은 존재

**네트워크 계층**

하드웨어와 직접적으로 맞다아 있는 계층

OSI 7계층의 물리, 데이터링크 계층의 역할을 수행한다

Node-Node로 연결되는 구조에서 MAC주소를 이용하여 연결한다

**인터넷 계층**

IP를 담당하는 계층

목적지와 출발지의 정보를 가지고 있고 IP이기 때문에

패킷 손실에 대해 생각치 않고 빠르게 전달하는 것에만 목적을 가진다

**전송 계층**

TCP/UDP를 담당하는 계층

TCP는 위 설명대로 순서대로 빠진 것 없이 전달하는 것이 목표이다

**응용 계층**

HTTP / DNS 를 담당하는 계층

서버, 클라이언트의 응용 프로그램이 해당 계층에서 동작한다

브라우저같은 서비스가 이에 해당

![스크린샷 2026-02-21 190657.png](%EC%8A%A4%ED%81%AC%EB%A6%B0%EC%83%B7_2026-02-21_190657.png)

각 계층이 따로 활동하는 것이 아닌 데이터가 4~1 / 1~4로 이동하면서 각자 목표에 맞게 패킷을 씌운 뒤 링크 계층을 통해 연결되는 구조이다.

> **PDU(프로토콜 데이터 단위)**
> 

각 계층에서 송수신되는 메세지의 단위

각 계층을 거치면서 단위의 이름이 달라진다

위 설명처럼 상위 계층에서 쌓인 헤더에 현재 계층에서 쌓은 헤더를 더한 것이

현재 PDU이다.

```java
현재 계층의 PDU = 상위 계층 데이터 + 현재 계층의 프로토콜 헤더(및 트레일러)
```

**데이터**

7계층에서 응용 계층 위로는 데이터라고 불린다

**세그먼트(전송 계층)**

데이터를 받은 전송 계층은 아래 정보들을 추가해 그룹화한다.

이때 세그먼트라고 불리운다

- 발신지 포트 : 발신하는 애플리케이션의 포트(프로그램 주소)
- 목적지 포트 : 수신해야할 애플리케이션의 포트
- 순서 번호 : 순서를 붙여서 어긋나는 경우 바로 고칠 수 있도록 한다
- 오류 검출 코드 : 체크섬을 통해 오류가 있는 확인 후 있다면 폐기후 복구 절차를 밟는다.

**패킷(인터넷 계층)**

세그먼트에서 네트워크 계층(IP 헤더)을 포함한 것을 패킷이라 부른다

- 발신지 IP : 패킷의 발신자 주소
- 목적지 IP : 패킷의 수신자 주소
- 서비스 요청 : 우선순위를 설정할 수 있다

**프레임(네트워크 엑세스 계층)**

IP패킷에 전송 가능하게끔 만든 것

- 목적지 MAC 주소
- 출발지 MAC 주소

**비트(물리적 표현)**

최종적으로 비트로 변환하여 실제 이동한다.
