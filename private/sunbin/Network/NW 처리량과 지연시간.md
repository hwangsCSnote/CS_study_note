# NW 처리량과 지연시간

> **네트워크 기초**
> 

네트워크란 노드(Node)와 링크(Link)가 서로 연결되어 리소스를 공유하는 집합을 말한다.

노드 : 서버, 라우터, 스위치 등

링크 : 연결매체, WIFI나 LAN

> **네트워크 처리량**
> 

링크를 통해 전달되는 단위 시간당 데이터양

보통 트래픽 처리량을 나타낸다

아래 요인들을 통해 처리량이 영향을 받게 된다.

- 트래픽
- 대역폭
- 패킷 손실
- 하드웨어 스펙

**패킷**

물리적인 네트워크를 통해 데이터가 전송될 때, 큰 데이터를 한번에 보내기보다 짧게 끊어서 보내는 것이 효율적이다. 이때 데이터를 잘게 쪼갠 것을 패킷이라고 부른다. 패킷은 아래 구조로 이루어져있다

- 헤더
    
    패킷의 헤더에는 출발지, 도착지, 패킷의 크기 등이 포함된다
    
    어디로 가야하는 지 결정하는 데 사용된다
    
- 데이터
    
    패킷의 본문에 해당하고 데이터의 정보가 들어가 있다
    
- 푸터
    
    데이터 무결성 확인을 위해 어디에서 변경된건지 확인하기 위해 사용된다
    

> **대역폭**
> 

주어진 시간 동안 네트워크 연결을 통해 흐를 수 있는 최대 비트 수 즉, 이론상 가능한 최대 전송 속도

단위는 bps이고, 기본적으로 손실, 지연, 혼잡을 모두 반영한다

> **지연 시간**
> 

요청이 처리되는 시간, 메시지가 두 장치를 왕복하는 데 걸린 시간

**지연 시간을 정하는 요소**

- 통신거리
    
    네트워크는 거리가 멀수록 멀리 이동해야하기 때문에 지연시간이 길어질 수밖에 없다
    
- 네트워크 정체
    
    트래픽이 몰리는 경우 특정 노드에서 정체되는 현상이 발생할 가능성이 증가한다. 이때 패킷이 기다리다가 다른 루트를 이용하는 경우도 생긴다
    
- 보안 관련 프로토콜
    - SSL/TLS
        
        데이터 통신의 보안을 위해 사용된다. 서버와 클라이언트가 서로 신원을 확인한다.
        
        통신 자체를 보호하는 기술이다. http가 이에 해당한다
        
    - IPsec
    - SSH
        
        원격 터미널 접속과 파일 전송을 안전하게 수행하기 위한 프로토콜
        
        원격 서버에 접속할 때 사용자 인증 등 보안 확인 후 원격으로 제어하는 것이 SSL과의 차이점이다.
        

ping 명령어를 사용하면 왕복시간(RTT)를 확인 할 수 있고, 밀리초로 표시되며 네트워크에서 데이터를 전송하는 데 걸리는 시간을 파악할 수 있다.

**RTT는 어떻게 측정되나?**

ping 또는 다양한 네트워크 진단 도구를 활용하여 측정할 수 있다

- 프롬프트에 ping 입력
- 테스트하려는 IP주소 또는 호스트 이름 입력

데이터 패킷을 각 대상에게 전달하고 왕복시간을 보고한다

당시 네트워크 상태나 측정에 사용되는 도구에 따라 RTT가 달라질 수 있기 때문에 RTT를 추정하는 것이 어렵다

일반적으로 RTT는 100ms 미만이 최적이다. 200ms이상이면 성능저하, UX감소를 생각해야한다.

**RTT(지연시간)를 줄일 수 있는 방법은 무엇인가?**

CDN을 사용하여 줄일 수 있다

CDN은 사용자와 더 가까운 위치에서 콘텐츠를 캐시하고 고가용성(오랫동안 정상운영되는 능력)을 제공하는 전략적으로 배치된 서버이다.

CDN은 캐싱, 로드 분산, 확장성을 통해 RTT를 줄인다.

- 캐싱
    
    동일한 데이터의 복사본을 저장하여 최종 사용자와 더 가까운 위치에 캐싱해놓는다
    
    지리적으로 멀리 떨어져 있다면 사용자가 요청했을 때 데이터를 전송하고 CDN에게도 전송해놓는다
    
    그런 후 동일한 요청을 보냈을 때 CDN을 통해서 전송해준다
    
- 로드 분산
    
    여러 엣지 서버로 캐싱을 전달한다
    
    로드 밸런서와의 차이점은 로드 밸런서는 내 서버 내부의 분산이라면 해당 로드 분산은 글로벌 지역에게 캐싱을 보내주는 형태이다.
    
- 확장성
    
    여러 엣지 서버에게 CDN을 통해 트래픽을 분산하여 오리진 서버에게 들어오는 요청을 걷어낸다
    

**처리량과의 관계**

성공적으로 전달된 데이터의 양을 의미하는 처리량은 대역폭과 지연시간에게 영향을 받는다

대역폭이 아무리 넓어도 지연시간이 길다면 처리량은 낮아질 수 있다

> **트래픽**
> 

특정 시점에 링크 내에 흐르는 데이터의 양, 서버를 통해 최종 사용자에게 전달된 데이터의 양을 의미한다

처리량과의 차이점은 트래픽은 단순히 흐르는 데이터의 양이고, 처리량은 성공적으로 전송된 점이다.

대역폭은 몇차선인지, 처리량은 성공적으로 지나간 차량을, 트래픽은 당장 얼마의 차가 지나가는 지를 의미한다.
