# 데이터베이스 설계 실습

화장품 웹사이트 DB를 설계함으로써 데이터 모델링의 이해를 돕는다

데이터 모델링의 이론 공부에 집중하기 보다 설계에 적합한 방식을 찾는 목적으로 작성한 문서이다.

### 데이터 모델링이란?

데이터 모델링은 정보시스템 구축의 대상을 분석하고 이해하여 약속한 표기법으로 표현하는 것을 의미한다.

데이터 모델링은 데이터베이스 설계의 핵심 과정이다.

### 데이터 모델링 순서

**1. 요구사항 수집 및 분석**

구축하려는 대상의 요구사항을 수집하여 어떠한 업무를 데이터화 할 것인지 분석한다.

의뢰인과 함께 요구사항을 분석할 경우 UI을 같이 보는 것이 효과적이다.

**2. 개념적 모델링**

분석한 데이터들을 추출하고 관계를 구상하여 데이터 구조를 이해할 수 있도록 한다.

- 엔터티 추출
- 엔터티 간 관계 정의
- 관계의 책임 정의

**3. 논리적 모델링**

구상한 데이터 간 구조를 구체화한다. Key, 속성, 관계 등을 표시한다. 정규화 활동을 수행한다.

**4. 정규화의 이해**

- 제 1정규화
    
    속성의 값을 하나만 가지도록 한다.
    
    테이블의 컬럼이 원자값을 갖도록 테이블을 분해한다
    
- 제 2정규화
    
    기본키의 부분집합이 결정자가 되지 않아야한다
    
    완전 함수 종속을 의미한다
    
    속성 A,B로 이루어진 복합키가 있다고 가정했을 때
    
    속성 B가 다른 테이블의 결정자(기본키)가 된다면
    
    테이블을 정규화해야한다.
    
- 제 3정규화
    
    이행적 함수 종속이 일어나면 안된다.
    
    A,B,C 속성이 존재할 때 A → B, B → C와 연관이 되어있다면 C를 변경할 때 A,B 둘 다 변경해야한다
    
    효율적이지 못한 DB설계이다.

**5. 물리적 모델링**

데이터를 관리할 데이터베이스를 선택하고 실제 테이블을 만드는 작업이다. SQL코딩을 통해 DB를 완성하는 단계이다.
