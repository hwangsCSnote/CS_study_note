# 네트워크의 기초  

**네트워크**(Network)  
> 컴퓨터나 기타 기기들이 리소스를 공유하거나 데이터를 주고 받기 위해 유선 또는 무선으로 연결된 통신체계  

- 흔히 **노드**(Node)와 **링크**(Link)가 연결된 형태로 표현한다.  
- **노드**(Node) : 서버, 라우터, 스위치 등 네트워크 장비  
- **링크**(Link) : 유선 / 무선 연결

### 처리량(Throuthput)  
> 링크(Link) 내에서 성공적으로 전달된 데이터의 양, 얼만큼의 **트래픽을 처리**했는지 나타냄  

<img width="366" alt="image" src="https://github.com/user-attachments/assets/1f5bdf85-1685-4a94-a72e-d22854a2ea4b" />


- **bps**(Bits Per Second) 단위를 사용 (= 초당 전송/수신 되는 비트 수)  
- 처리량은 **트래픽**, **대역폭** 등에 영향을 받음  

**대역폭**  
> 주어진 시간 동안 네트워크 연결을 통해 흐를 수 있는 최대 비트 수  

<br>
<br>

### 지연 시간(Latency)  
> 요청이 처리되는 시간, 하나의 메시지가 두 장치를 왕복하는데 걸린 시간  

- 매채 타입(무선/유선), 패킷 크기, 라우터의 패킷 처리 시간 등에 영향을 받는다.  

<br>
<br>

### 네트워크 토폴로지(Network Toology)  
> 노드(Node)와 링크(Link)가 배치되어 있는 방식과 연결 형태  

<img width="486" alt="image 2" src="https://github.com/user-attachments/assets/3d2e3852-f577-4b00-bf3e-8d84136f66a1" />

**트리형**(Tree)  
> 계층형 토폴로지 라고도 부르며 **트리 형태**로 배치한 네트워크 구성  
- 노드의 추가, 삭제가 쉬우며 특정 노드에 트래픽이 집중될 때 하위 노드에 영향을 끼칠 수 있음  

**버스형**(Bus)  
> **중앙 통신 회선 하나에 여러 개의 노드가 연결**되어 공유하는 네트워크 구성  
- 주로 근거리 통신망(LAN)에서 사용한다.  
- 설치 비용이 적고 신뢰성이 우수하며 중앙 통신 회선에 노드를 추가하거나 삭제하기 쉽다.  
- **스푸핑**(Spooping)이 가능하다는 문제점이 있다.  

**스타형**(Star)  
> **중앙에 있는 노드에 모두 연결**되어있는 네트워크 구성  
- 노드를 추가하거나 에러를 탐지하기 쉽다.  
- 패킷의 충돌 가능성이 적다.  
- 중앙 노드에 장애가 발생하면 전체 네트워크를 사용할 수 없다.  
- 설치 비용이 비싸다.  

**링형**(Ring)  
> 각각의 노드가 양 옆의 두 노드와 연결하여 **전체적으로 고리처럼 하나의 연속된 길을 통신**하는 망 구성  
- 노드의 수가 증가해도 네트워크 상의 손실이 거의 없고 충돌 가능성도 적다.  
- 네트워크 구성 변경이 어렵고 회선에 장애 발생 시 전체 네트워크에 영향을 끼친다.  

**메시형**(Mesh)  
> 망형 토폴로지라고도 하며, **그물망처럼 연결**되어 있는 구조  
- 하나의 단말 장치에 장애가 발생해도 여러 개의 경로가 존재하기 때문에 네트워크를 계속 사용할 수 있다.  
- 트래픽도 분산 처리가 가능하다.  
- 하지만 노드의 추가가 어렵고, 구축 비용과 운영 비용이 비싸다.  

<br>

### 병목현상(Bottleneck)  
> 병의 몸통보다 목 부분 지름이 좁아 물이 상대적으로 천천히 쏟아지는 것에 비유하여  
> 전체 시스템의 성능이나 용량이 하나의 구성 요소로 인해 제한 받는 현상  

- 병목현상이 발생할 경우, 네트워크가 어떤 토폴로지를 갖는지, 어떤 경로로 이루어져 있는지 알아야 바르게 해결할 수 있다.  

<br>

### 네트워크 분류  
네트워크 규모에 따라 **LAN**(Local Area Network), **MAN**(Metropolitan Area Network), **WAN**(Wide Area Network) 로 나뉜다.  

**LAN**(Local Area Network)  
> 근거리 통신망을 의미하며 **같은 건물이나 캠퍼스** 등 좁은 공간에서 운영  
- 전송 속도가 빠르고 혼잡하지 않다.  

**MAN**(Metropolitan Area Network)  
> 대도시 지역 네트워크를 나타내며 **도시** 같은 넓은 지역에서 운영  
- 전송 속도는 평균적이며 LAN보다는 더 혼잡하다.  

**WAN**(Wide Area Network)  
> 광역 네트워크를 의미하며 **국가 또는 대륙** 같은 더 넓은 지역에서 운영  
- 전송 속도는 느리며 MAN보다 더 혼잡하다.  

<br>

### 네트워크 성능 분석 명령어  

일반적인 네트워크 병목 현상의 주된 원인  

1. 네트워크 대역폭  
2. 네트워크 토폴로지  
3. 서버 CPU, 메모리 사용량  
4. 비효율적인 네트워크 구성  

→ 네트워크 성능 분석 필요

### 네트워크 프로토콜  
> 다른 장치들끼리 데이터를 주고 받기 위해 설정된 공통된 인터페이스  

- IEEE 또는 IETF 라는 표준화 단체에서 설정  
- **IEEE802.3** : 유선 LAN 프로토콜  
- **HTTP** : 웹 브라우저 통신 프로토콜  
- **FTP** : 파일 전송용 프로토콜  
- **DNS** : 도메인 IP 매핑용 프로토콜
