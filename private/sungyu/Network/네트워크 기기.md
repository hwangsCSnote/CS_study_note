# 네트워크 기기  

### 네트워크 기기의 처리 범위  
- **애플리케이션 계층** : L7 스위치  
- **인터넷 계층** : 라우터, L3 스위치  
- **데이터 링크 계층** : L2 스위치, 브리지  
- **물리 계층** : NIC, 리피터, AP  

→ 상위 계층 기기는 하위 계층을 처리할 수 있지만 반대 작업은 불가능하다.  


**스위치**  
> 컴퓨터 네트워크에서 여러 장비들을 연결하고 데이터를 전송하는 장비

### 애플리케이션 계층 : L7 스위치  
> L7 스위치는 로드벨런서라고도 하며, 서버의 부하를 분산하는 기기  

- 클라이언트로부터 오는 요청들을 여러 서버로 나누는 역할을 한다.  
- URL, 서버, 캐시, 쿠키들을 기반으로 **트래픽을 분산**한다.  
- 응용 프로그램 수준의 **트래픽 모니터링**이 가능하다.  
- 로드벨런서로는 **L7 스위치**, **L4 스위치**가 있다.  
- 장애가 발생한 서버를 트래픽 분산 대상에서 제외하기 위해 헬스 체크(Health Check)를 이용한다.  

※ 헬스 체크(Health Check)  
: TCP, HTTP 등 다양한 방법으로 서버에 반복적인 요청을 전송하여 서버의 상태가 정상적인지 판단  

**L4 스위치 vs L7 스위치 차이**  
- 전송 계층 vs 애플리케이션 계층  
- IP, Port 번호를 기반으로 패킷 전송 vs HTTP, HTTPS, FTP 등 URI에 따라 트래픽을 분배  
- NLB(Network Load Balancer) vs ALB(Application Load Balancer)  

<br>

### 인터넷 계층 : 라우터, L3 스위치  

**라우터**(Router)  
- 여러 개의 네트워크를 연결, 분할, 구분시켜주는 역할  
- 다른 네트워크에 존재하는 장치끼리 서로 데이터를 주고받을 때 패킷 소모를 최소화하고 경로를 최적화  

**L3 스위치**  
- L2 스위치 기능(MAC 주소, 이더넷 프레임) + 라우팅 기능(IP 주소, IP 패킷)  
- 내부 망(LAN)의 고속 라우팅에 특화  

<br>

### 데이터 링크 계층 : L2 스위치, 브리지  

**L2 스위치**  
> 장치들의 MAC 주소를 MAC 주소 테이블을 통해 괸리하며, 패킷 전송을 담당  

- L3 스위치와 달리 IP 주소 기반의 라우팅이 불가능하다.  
- 단순히 패킷의 MAC 주소를 읽어 스위칭 하는 역할을 담당  

**브리지**(Bridge)  
> 두 개의 근거리 통신망(LAN)을 상호 접속할 수 있도록 하는 통신망 연결장치  

- MAC 주소를 기반으로 두개의 LAN을 연결하고 트래픽을 관리  
- 통신망 범위를 확장하고 서로 다른 LAN을 하나의 통신망으로 구축할 때 사용  

<br>

### 물리 계층 : NIC, 리피터, AP  

**NIC**(Network Interface Card)  
> 흔히 LAN 카드 라고 부르는 장치  

![download](https://github.com/user-attachments/assets/954c692c-8f9e-4dc6-a011-d5928b3f30e6)

- 네트워크와 빠른 속도로 데이터를 송수신 할 수 있도록 컴퓨터 내에 설치하는 확장 카드  


**리피터**(Repeater)  
> 수신하는 신호를 증폭하여 다른 쪽으로 전달하는 장치  

- 리피터를 통해 패킷이 더 멀리 이동할 수 있다.  
- 광케이블이 보급되며 현재는 잘 사용되지 않는다.  


**AP**(Access Point)  
> 패킷을 복사하는 기기  

- 유선 LAN을 연결하여 무선 LAN 신호로 변환하여 무선으로 네트워크 연결이 가능하다.
