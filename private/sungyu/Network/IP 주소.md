# IP 주소  

### ARP(Address Resolution Protocol)  
> IP 주소로부터 MAC 주소를 구하는 프로토콜  

- **IP 주소** (가상 주소 / 논리적 주소)  →  **MAC 주소** (실제 주소 / 물리적 주소)  
  1. **ARP Request 브로드캐스트**를 보낸다. (IP 주소값)  
  2. 해당 주소에 맞는 장치에서 **ARP Reply 유니캐스트**를 통해 반환한다. (MAC 주소값)  

<br>

  ※ 브로드캐스트 : 네트워크에 연결된 모든 호스트에 데이터를 전송하는 방식  
  ※ 유니캐스트 : 하나의 네트워크 목적지에 1:1로 데이터를 전송하는 방식  

- **MAC 주소** (실제 주소 / 물리적 주소)  →  **IP 주소** (가상 주소 / 논리적 주소)  
  : **RARP**(Reverse Address Resolution Protocol)  

<br>
<br>

### 홉바이홉 통신(hop)  
> IP 주소를 통해 통신하는 방식  
> → 데이터 패킷이 네트워크 장비(라우터)를 거쳐 전송되는 과정을 의미  


**라우팅** (Routing)  
> 라우터(Router)가 라우팅 테이블을 참조해 최적의 경로를 설정하고, 데이터를 전달하는 과정  


**라우팅 테이블** (Routing Table)  
> 라우터에 들어있는 목적지 정보들과 목적지로 가기 위한 방법이 들어있는 리스트  

- `netstat -r`  을 통해 라우팅 테이블 정보를 확인할 수 있다.  



**게이트웨이** (Gateway)  
> 네트워크 간 통신을 위한 데이터를 송수신하는 관문  

- 일반적으로 해당 네트워크 서브넷의 첫 번째(`X`.`X`.`X`**.1**) 또는 마지막(`X`.`X`.`X`**.254**) 주소를 사용한다.  

<br>

### IP 주소체계  

**IPv4**  
> 32비트를 8비트 단위로 나눠 표기  

ex) `3.38.166.214`  


**IPv6**  
> 64비트를 16비트 단위로 나눠 표기  

ex) `::ffff:0326:a6d6`  

<br>

**클래스 기반 할당 방식**  
> IP 주소 체계를 규모(호스트 수)에 따라 A, B, C, D, E 다섯 개의 클래스로 구분하는 방식  


- **A 클래스** : 대규모 네트워크용  
  - ( `0 ~ 127` )  
  - 0.0.0.0 ~ 127.255.255.255  
  - **0**0000000.00000000.00000000.00000000 ~ **0**1111111.11111111.11111111.11111111  

- **B 클래스** : 중간 규모 네트워크용  
  - ( `128 ~ 191` )  
  - 128.0.0.0 ~ 191.255.255.255  
  - **10**000000.00000000.00000000.00000000 ~ **10**111111.11111111.11111111.11111111  

- **C 클래스** : 소규모 네트워크용  
  - ( `192 ~ 223` )  
  - 192.0.0.0 ~ 223.255.255.255  
  - **110**00000.00000000.00000000.00000000 ~ **110**11111.11111111.11111111.11111111  

- **D 클래스** : 멀티캐스트용  
- **E 클래스** : 연구/실험 및 예비용  

※ 네트워크에서 가장 첫 번째 주소는 (`X`.`X`.`X`**.0**) 네트워크 구분용 주소로 사용된다.  
※ 네트워크에서 가장 마지막 주소는 (`X`.`X`.`X`**.255**) 브로드캐스트용 주소로 사용된다.  

→ 실질적으로 할당할 수 있는 호스트 주소의 범위는 ( `X`.`X`.`X`**.1** ~ `X`.`X`.`X`**.254** ) 이다.

**DHCP**(Dynamic Host Configuration Protocol)  
> IP 주소 및 기타 통신 매개변수를 자동으로 할당하기 위한 네트워크 관리 프로토콜  

- 직접 **수동**으로 연결할 IP 설정 가능 (복잡하고 전문 지식이 필요)  
![](%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA%202026-03-14%2013.29.07.png)<!-- {"width":283} -->

- 많은 라우터와 게이트웨이 장비에 DHCP 서버가 내장되어있다.  
- 사용자의 MAC 주소를 기반으로 라우터(공유기)를 통해 DHCP 요청을 보내며, IP를 자동으로 할당받는다.  
![](%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA%202026-03-14%2013.39.11.png)<!-- {"width":654} -->
- 사용자 기기에 내장된 DHCP 클라이언트에서 **자동으로 정보를 입력**하고 네트워크를 연결해준다.  
- 공유기 소프트웨어를 통해 DHCP 설정을 변경할 수 있다.

**NAT**(Network Address Translation)  
> 패킷의 IP 주소 정보를 수정하여 IP 주소를 다른 주소로 매핑하는 방법  

- 일반적으로 사설 네트워크(Private Network)에 속한 여러 개의 호스트가 하나의 공인 IP(Public IP) 주소를 사용하여 인터넷에 접속하기 위해 NAT를 사용한다.  
![](%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA%202026-03-14%2018.50.18.png)<!-- {"width":613} -->
Reference : https://www.youtube.com/watch?v=ctY8VtRd7cc  

1. LAN에 접속한 사용자 기기에서 외부망 연결을 요청  
2. 라우터(공유기)에서 요청한 사설 IP(Private IP)를 기록  
3. 외부망 연결을 위해 공인 IP(Public IP) 로 변환 후 요청을 전달  
4. 추후 기록했던 사설 IP(Private IP)로 연결 및 데이터 전송  

**효과**  
- 내부망(LAN)에서 사용하는 IP 주소와 외부망(WAN)에 드러나는 IP 주소를 다르게 유지할 수 있기 때문에 IP 노출에 대한 보안을 유지할 수 있다.  
- 여러 명이 동시에 인터넷에 접속하는 경우, 실제 접속하는 호스트 수에 따라 접속 속도가 느려질 수 있다.
