### 조인(Join)  
> 데이터베이스에서 두 개 이상의 테이블을 연결하여 하나의 결과물(테이블)을 만드는 것  

- 데이터베이스에서 테이블을 분리하여 관리하며 **데이터 중복을 줄이고, 일관성을 유지하기 위해** 사용  
- MySQL 에서는 **JOIN** 쿼리, MongoDB 에서는 **lookup** 쿼리로 처리  
- 대표적으로 **내부 조인**(INNER JOIN), **외부 조인**(OUTER JOIN : REFT, RIGHT, FULL)이 있다.

1. **내부 조인**(INNER JOIN)  
> 두 테이블 간 일치하는 튜플(Tuple)들로 결과 테이블을 생성  

- 일반적으로 JOIN을 이야기할 때, INNER JOIN을 의미한다.  
- ’=‘ , ‘<>’, ‘>’, ‘<‘ 등 다양한 비교 연산자를 사용해 조건을 지정할 수 있다.  
- 쿼리문에서 (INNER)를 생략하여 표기할 수 있다. - **(INNER) JOIN**

2. **외부 조인**(OUTER JOIN)  
> 두 테이블 간 공통된 값이 아닌 튜플(Tuple)들도 포함하여 결과 테이블을 생성  

- 공통되지 않은 값의 범위에 따라 LEFT OUTER JOIN, RIGHT OUTER JOIN, FULL OUTER JOIN 으로 나뉜다.  

  1. **LEFT OUTER JOIN**  
  > 두 테이블 간 공통된 값 + 왼쪽 테이블 모든 값 (빈 값은 NULL 반환)  

  - 쿼리문에서 (OUTER)를 생략하여 표기할 수 있다. : **LEFT (OUTER) JOIN**

3. **자연 조인**(Natural JOIN)  
> 두 테이블에서 **같은 속성명**을 가지는 속성(Attribute)에 대해 동등 조인(‘=‘)을 수행  
> (조인 조건은 명시하지 않음, 자동으로 설정)  

- INNER JOIN, OUTER JOIN 모두 적용 가능하다.  
- 동일한 속성명은 결과 테이블 내에서 한 번만 표기한다.  

```sql
SELECT *
FROM Table_A, NATURAL (INNER) JOIN Table B

--- FROM Table A (INNER) JOIN Table B USING(key)
--- FROM Table A (INNER) JOIN Table B ON Table A.key = Table B.key
```

```sql
SELECT *
FROM Table A, NATURAL LEFT (OUTER) JOIN Table B

--- FROM Table A LEFT (OUTER) JOIN Table B USING(key)
--- FROM Table A LEFT (OUTER) JOIN Table B ON Table A.key = Table B.key
```

4. **교차 조인**(Cross JOIN)  
> 두 테이블에서 튜플 쌍으로 만들 수 있는 **모든 조합**(Cartesian product)을 결과로 반환  

- MySQL에서 CROSS JOIN에 **ON(or USING) 등 조건을 함께 사용한다면** INNER JOIN 으로 동작한다.  
- MySQL에서 INNER JOIN에 **ON(or USING) 등 조건 없이** 사용한다면 CROSS JOIN 으로 동작한다.
