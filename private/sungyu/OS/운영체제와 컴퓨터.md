# 운영체제와 컴퓨터  

### 중앙처리장치(Central Processing Unit, CPU)  
> 산술논리연산장치(ALU), 제어장치, 레지스터로 구성되어 컴퓨팅 장치의 두뇌 역할을 담당

**제어장치**(Control Unit, CU)  
>  프로세스 조작을 지시하고 제어하는 CPU의 핵심 부품  

- 입출력장치 간 통신을 제어  
- 명령어들을 읽고 해석  
- 데이터처리를 위한 순서를 결정  

<br>

**레지스터**(Register)  
> 여러 비트 정보를 저장하는 회로로서, CPU 내부에 위치한 매우 작고 빠른 임시기억장치  

- 일반적인 메모리보다 연산 속도가 수십 배에서 수백 배까지 빠름  
- CPU에서는 데이터를 저장할 방법이 없기 때문에 레지스터를 거쳐 데이터를 전달  

<br>

**산술논리연산장치**(Arithmetic Logic Unit, ALU)  
> 덧셈, 뺄셈 같은 두 숫자의 산술 연산과 배타적 논리합, 논리곱 같은 논리 연산을 계산하는 회로  

<br>

※ **CPU 연산처리 과정**  
1. **제어장치**가 메모리(RAM)에서 계산할 값을 로드  
2. 메모리(RAM)에서 가져온 값을 **레지스터에 저장**  
3. 레지스터에 저장한 값을 **산술논리연산장치(ALU)에서 연산처리**  
4. 계산된 결과를 레지스터에서 **메모리(RAM)로 반환 및 저장**  

<br>

**인터럽트**(Interrupt)  
> 프로그램 실행 중 예기치 않은 상황이나 입출력, 오류 등 긴급한 이슈 발생 시, 실행 중인 작업을  
> 중단하고 발생된 상황을 우선 처리하는 과정  

- **하드웨어 인터럽트**  
  : 키보드, 마우스, 디스크 등 외부 장치에서 발생하는 인터럽트  


- **소프트웨어 인터럽트**  
  : 주로 트랩(Trap)이라 부르며, 프로세스 오류 등으로 프로세스가 시스템콜을 호출할 때 발동  
  - 주로 예외 상황 또는 시스템콜을 사용해야하는 경우 인터럽트 발생  

<br>
<br>

### 메모리(Random Access Memory, RAM)  
> 데이터의 상태, 명령어 등을 기록하는 장치  

<br>

### 타이머(Timer)  
> 특정 프로그램에 시간제한을 두고, 작업에 필요한 일정 시간 초과 시 제한  

<br>

### 운영체제(Operating System, OS)  
> 하드웨어와 응용 프로그램 사이 연결 다리 역할을 하며, 시스템 자원(CPU, 메모리, 입출력 장치 등)을  
> 관리하고 효율적으로 배분하는 시스템 소프트웨어

**시스템콜** 
> 운영체제가 커널(Kernel)에 접근하기 위한 인터페이스로, 프로그램이 운영체제의 서비스를 받기 위해  
> 커널 함수 호출 시 사용

<br>

**커널**(Kernel)  
> 시스템콜 인터페이스를 제공하며 보안, 메모리, 프로세스, 파일 시스템, I/O 디바이스, I/O 요청 관리 등   
> 운영체제의 중추적인 역할을 담당  

<br>

- **modebit**  
  : 시스템콜이 작동될 때, 유저 모드와 커널 모드를 구분하기 위한 플래그 변수  

- **유저 모드** (modebit = 1) vs **커널 모드** (modebit = 0)  
  : 사용자가 접근할 수 있는 영역에 제한을 두어 컴퓨터 자원에 함부로 침범하지 못하게 막기 위한 장치로  
  커널 모드에서만 컴퓨터의 자원에 접근하고 제어할 수 있다.  

<br>

※ **운영체제의 역할**  
1. **CPU 스케줄링 및 프로세스 관리**  
   : CPU 소유권을 어떤 프로세스에 할당할지, 프로세스의 생성과 삭제, 자원 할당 및 반환을 관리  
2. **메모리 관리**  
   : 한정된 메모리를 어떤 프로세스에 얼마나 할당할지 관리  
3. **디스크 파일 관리**  
   : 디스크 파일을 어떤 방법으로 보관할지 관리  
4. **I/O 디바이스 관리**  
   : I/O 디바이스들인 마우스, 키보드와 컴퓨터 간에 데이터를 주고받는 것을 관리
